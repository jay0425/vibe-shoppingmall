# 기능 개발 순서

1. 스키마 제작
2. crud 라우터, 로직 구현
3. fe - 페이지 디자인
4. 서버와 프론트 연결 로직
5. 테스트

# 스키마

## 유저 스키마

email 필수
name 필수
password 필수
user_type 필수, customer (customer, admin)
address 필수 아님

## 상품 스키마

sku (상품 id값. 유니크해야 함.)
상품 이름
상품가격
카테고리 (상의, 하의, 액세서리)
이미지 => cloudinary (이미지를 string 값으로)
설명 필수값 아님

## 로그인

1. 이메일, 비밀번호(로그인 실패/성공 처리, 잘못된 정보 입력 시 에러 메시지 출력)
2. 자동 로그인 기능
3. 토큰 발급 : 유저 권한이 필요한 작업 시 로그인 매번 요청하지 않고 인증정보로 작업 가능하도록 구현.
4. 로그아웃

5. 스키마 제작
6. 로그인 라우터, 로직 구현
7. fe - 페이지 디자인
8. 서버와 프론트 연결 로직
9. 테스트
